Hydra Quick Reference Guide\n\n1. Basic Syntax:\nhydra [[[-l LOGIN|-L FILE] [-p PASS|-P FILE]] | [-C FILE]] [-e ns|r|s] [-o FILE] [-t TASKS] [-M FILE [-T TASKS]] [-w TIME] [-W TIME] [-f] [-s PORT] [-x MIN:MAX:CHARSET] [service://server[:port]/path] [-a SERVICE]
\n2. Flags:\n-l: login name (single)\n-L: login list file\n-p: password (single)\n-P: password list file\n-C: colon separated \'login:password\' file\n-e: extension (-e ns| r| s)\n-o: write output to a file\n-t: tasks to run parallel (default: 16)\n-M: server list for parallel attacks\n-T: tasks to run per server (default: 16)\n-w: wait time for response (default: 30)\n-W: maximum wait time for response (default: 30)\n-f: exit after the first login/password pair is found\n-s: connect to the specified port (default: 80)\n-x: brute force with a predefined set of characters
\n3. Examples:\n- Brute force ssh using a single login and password:\nhydra -l username -p password ssh://192.168.0.1\n\n- Brute force ssh using a list of logins and a single password:\nhydra -L /path/to/logins.txt -p password ssh://192.168.0.1\n\n- Brute force ssh using a single login and a list of passwords:\nhydra -l username -P /path/to/passwords.txt ssh://192.168.0.1\n\n- Brute force ssh using a list of logins and a list of passwords:\nhydra -L /path/to/logins.txt -P /path/to/passwords.txt ssh://192.168.0.1
